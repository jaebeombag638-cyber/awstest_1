# 영양제 추천 웹서비스 (awstest1) 프로젝트 문서

## 개요

사용자가 간단한 건강 설문을 작성하면 AI가 맞춤 영양제 카테고리를 추천하고, 약국 제품과 온라인 마켓 제품을 비교한 뒤 최종 추천까지 해주는 한국어 웹 서비스. 비상업용 연습 프로젝트.

---

## 기술 스택

| 항목 | 기술 |
|---|---|
| 프레임워크 | Next.js 16.2.9 (App Router, TypeScript) |
| 스타일 | Tailwind CSS v4.3.1 |
| AI 추천 | OpenAI GPT-4o (`gpt-4o`) |
| 지도 | Kakao Maps JavaScript SDK + Kakao Local REST API |
| 런타임 | Node.js (로컬 개발 서버) |

---

## 서비스 플로우

```
1. 건강 설문 (5문항)
       ↓
2. GPT-4o → 영양제 카테고리 3~4개 추천
       ↓
3. 카테고리 클릭 → 약국 제품 3개 vs 온라인 제품 3개 비교
       ↓
4. 각 제품 리뷰 확인 / 구매 링크 / 약국 찾기
       ↓
5. GPT-4o → AI 최종 추천 (약국 vs 마켓, 최선의 제품 1개)
       ↓
6. 약국 추천 시 → 카카오맵으로 근처 약국 표시
   마켓 추천 시 → 쿠팡 검색 링크
```

## 주요 기능 상세

### 건강 설문 (page.tsx)
- 5문항: 건강 고민(복수), 연령대, 식습관, 햇빛 노출, 증상(복수)
- 답변은 `sessionStorage`에 저장 후 추천 페이지로 이동

### AI 추천 (api/recommend)
- GPT-4o에 설문 답변 + 카테고리 목록 전달
- JSON 형식으로 카테고리 3~4개 + 추천 이유 반환
- `response_format: { type: 'json_object' }` 사용

### 제품 비교 (compare/[category])
- 약국 제품 (파란 컬럼) vs 온라인 제품 (주황 컬럼) 나란히 표시
- 각 카드: 별점, 리뷰 수, 특징 태그, 리뷰 토글
- 약국 카드마다 "근처 약국 찾기" 버튼 (AI 추천 결과와 무관하게 접근 가능)
- 온라인 카드마다 "구매하기" → 쿠팡 제품명 검색 결과 링크

### AI 최종 추천 (api/ai-final)
- GPT-4o가 약국 vs 온라인 중 winner 결정
- 반환값: winner, 이유, 약국/마켓 각 분석, 최선의 제품 1개, 주의사항

### 약국 찾기 (api/pharmacies + PharmacyFinder)
- 서버: 주소 입력 → Kakao 주소 검색 API → 좌표 변환 → 반경 2km 내 약국 키워드 검색
- 주소 검색 실패 시 키워드 검색으로 자동 폴백
- 클라이언트: Kakao Maps JS SDK로 지도 렌더링, 마커 클릭 시 정보창 표시

## 한계점

### 데이터 관련
- **리뷰가 가상 데이터**: 실사용자 리뷰가 아닌 샘플 텍스트. 실제 리뷰 연동은 크롤링/API 제휴 필요.
- **제품 정보가 정적**: 가격, 재고, 신제품 반영 안 됨. DB 연동 없이 코드에 하드코딩.
- **쿠팡 링크가 검색 결과**: 정확한 제품 페이지 URL이 아닌 제품명 검색 링크. 쿠팡 파트너스 API 없이는 정확한 링크 불가.

### 약국 찾기 관련
- **재고 확인 불가**: "근처 약국"을 보여줄 뿐, 해당 약국에 그 제품이 실제로 있는지는 알 수 없음. 약국별 실시간 재고 API가 존재하지 않음.
- **지도는 등록된 도메인에서만 작동**: 카카오 JS SDK 도메인 등록 필요. 로컬 개발 시 `http://localhost:3000` 등록해야 함.

### AI 추천 관련
- **근거 없는 추천**: GPT-4o가 실제 임상 데이터나 개인 혈액 검사 결과 없이 설문 답변만으로 추천. 의학적 조언 대체 불가.
- **제품 최신성 없음**: AI가 코드에 내장된 제품 목록만 보고 판단. 실시간 시장 데이터 반영 안 됨.

### 인프라 관련
- **로컬 전용**: 배포 없이 `localhost:3000`에서만 작동. 실 배포 시 Vercel/AWS 설정 + 환경변수 재설정 필요.
- **세션 기반 상태**: `sessionStorage` 사용으로 새로고침 시 설문 답변 유지되지만, 브라우저 종료 시 초기화.
- **API 키 관리**: `.env.local`에 키 보관. 배포 시 플랫폼 환경변수로 이전 필요.

### 확장 시 필요한 것들
- 실 사용자 리뷰: 리뷰 크롤링 파이프라인 또는 자체 리뷰 시스템
- 정확한 쿠팡 링크: 쿠팡 파트너스 API 제휴
- 약국 재고: 불가 (공개 API 없음) → 올리브영 등 체인으로 범위 한정하는 방향만 가능
- 제품 DB: 정기적으로 업데이트되는 영양제 DB 또는 관리자 페이지
- 의료 면책 고지: 실 서비스 시 "의학적 조언이 아님" 명시 필수
